]]> Don’t like the new Gutenberg editor? Since WordPress 5.0, the WP core has been updated with the Gutenberg editor.

The Gutenberg editor can be turned off thankfully, by adding a few lines of code to your theme’s function.php file.

/* Disable Gutenberg */

// disable for posts
add_filter('use_block_editor_for_post', '__return_false', 10);

// disable for post types
add_filter('use_block_editor_for_post_type', '__return_false', 10);

]]> So usually an image is outputted through a filepath in the ‘src’ attribute in an ‘img’ tag. Although we can manipulate the filepath in such a way we can actually recreate the image by getting the image’s data into base64 encoding. It helps protect outsiders from knowing your folder structure.

Note: IE7 and lower don’t support Data URIs Doing it this way also helps speed up page speed as there is no need to search for a url.

// A few settings
$image = 'cricci.jpg';

// Read image path, convert to base64 encoding
$imageData = base64_encode(file_get_contents($image));

// Format the image SRC:  data:{mime};base64,{data};
$src = 'data: '.mime_content_type($image).';base64,'.$imageData;

// Echo out a sample image
echo '<img src="', $src, '">';

]]> Make sure you first login to the admin of your WordPress site and navigate to Settings > General to update your site address to include ‘https’ in place of ‘http’.

Then open the .htaccess file in the root of your web directory and add this code to the top:

RewriteEngine On
RewriteCond %{HTTPS} !=on
RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]

]]> This bit of code updates the anchor point position when an anchor is called on the current page to 150px above the original element. NOTE: This does not work when navigating to a page with an anchor point.

        window.addEventListener("hashchange", function () {
	    window.scrollTo(window.scrollX, window.scrollY - 150);
	});

If you want to enable the anchor point position on page load, you will have to modify it like this:

	function offsetAnchor() {
	    if(location.hash.length !== 0) {
	        window.scrollTo(window.scrollX, window.scrollY - 150);
	    }
	}

	// Will find hash changes while on the page
	window.addEventListener("hashchange", offsetAnchor);

	// This is here so that when you enter the page with a hash,
	// it can provide the offset in that case too. Having a timeout
	// seems necessary to allow the browser to jump to the anchor first.
	window.setTimeout(offsetAnchor, 1);
